## 文档说明：怎么使用这个挑战库

### 这个库是什么

这是一个可以全年灵活取用的挑战题目集。每个挑战都是独立的——不需要做完某个才能做下一个，任何时候拿出来用都行。

### 怎么挑选合适的挑战

1. **先看目标阶段**：挑战和四个学习阶段对齐。学生还在阶段一就别拿阶段三的题，会挫败感爆棚。

2. **再看难度**：难度1-5级，1最简单，5最挑战。大多数学生日常做2-4级，留1和5给特殊场合——1级适合第一次接触某类任务，5级适合"已经很快做完了还想继续"的学生。

3. **看类别**：五个类别侧重不同的动机类型。有的学生天生喜欢创意类，有的对解决真实问题更有劲。

4. **小学版 vs 中学版**：同一个挑战有两个版本，不只是换个措辞，而是实质上的难度差异——中学版通常要求更多用户测试、更复杂的功能、更严格的成功标准。

### 各阶段工具提醒

- **阶段一**：仅用 Claude / ChatGPT 网页对话界面，不碰代码
- **阶段二**：Claude Artifacts 或 Cursor，用AI生成可运行网页，学生用语言描述修改
- **阶段三**：ClaudeCode / OpenCode，构建多模块应用，掌握 Debug 流程
- **阶段四**：自主发现问题，完整走产品设计流程，ClaudeCode 为主要工具

### 挑战01：用AI帮我写故事

**类别**：创意类
**目标阶段**：阶段一
**难度**：⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：45 分钟

**挑战目标**
用AI生成一个有固定主角的连载故事，至少三个章节，每章都有情节发展，学生全程通过对话引导AI写出自己满意的内容。

**所需材料**
- 电脑 + AI工具（Claude 或 ChatGPT 网页版）
- 故事记录本或文档（保存AI产出的故事）

#### 小学版（6-10岁）

**任务描述**
给AI设计一个有名字的主角（可以是动物、人、或者你发明的生物），告诉AI主角叫什么、住在哪、有什么特点。然后让AI帮你写三章故事，每章大概两三段，最后要有一个结局。

要求：
1. 让AI写完第一章后，先读一读。如果哪里不对或者不够有趣，告诉AI"这里改一下，我想要……"
2. 第二章的开头要提到第一章发生的事情（你要提醒AI"上一章主角做了什么，接着写"）
3. 写完三章后，给你的故事想一个标题

**成功标准**
- [ ] 故事有一个你自己设计的主角，有名字和至少一个特点
- [ ] 三章故事情节连贯，第二章接着第一章讲
- [ ] 你至少改过AI一次——把不满意的地方重新说清楚让它改
- [ ] 你能把第一章读给旁边的人听，对方听得懂发生了什么

**快速完成者的挑战加餐**
给故事加一个反派角色，让AI重新写一章新的"危机"——主角遇到了麻烦，但用自己的方法解决了它。

#### 中学版（11-15岁）

**任务描述**
设计一个系列故事，要有完整的世界观设定和至少两个主要角色（不只是主角，还要有一个配角或者对立角色）。让AI写四到五章，每章情节有起伏，不能每章结局都很顺利。

要求：
1. 开始前先告诉AI整体的故事背景（时代、地点、主要矛盾），然后让AI一章一章写
2. 每章写完后，评估：这章的节奏有没有问题？人物说话的方式符合他们的性格吗？把不满意的地方用具体的语言告诉AI
3. 最后一章必须解决整个故事的主要矛盾
4. 写完后为故事制作一份"人物档案"——每个主要角色一段话，说明他们的性格和在故事中的作用

**成功标准**
- [ ] 有至少两个主要角色，各有不同的性格，故事里能看出来
- [ ] 五章故事有明显的情节起伏，不是每章都平平淡淡
- [ ] 有至少三次"修改AI"的记录，每次能说出"改了什么，为什么改"
- [ ] 完成人物档案，每个角色都有具体的描述

**快速完成者的挑战加餐**
把故事最精彩的一章改写成剧本格式（有舞台提示、人物对话），然后找同学来"读剧本"，看看同学读出来的感觉和你想象的一不一样。

### 挑战02：我的兴趣百科

**类别**：创意类
**目标阶段**：阶段一
**难度**：⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：45 分钟

**挑战目标**
用AI为自己最感兴趣的一个话题创建一本专属"百科全书"，用清晰的分类整理AI帮你收集的知识，最终产出一份可以读给别人听的内容文档。

**所需材料**
- 电脑 + AI工具（Claude 或 ChatGPT 网页版）
- 一张纸，先写下"我想了解关于XXX的哪些问题"

#### 小学版（6-10岁）

**任务描述**
选一个你最感兴趣的主题（比如：恐龙、猫咪、足球、宇宙、甜点），让AI帮你写一本"主题百科"。

步骤：
1. 先想好这本百科想包含什么——写下至少5个你好奇的问题（比如："最大的恐龙是哪种？""恐龙为什么灭绝？"）
2. 把每个问题分别发给AI，让AI用简单易懂的方式回答
3. 如果AI的答案太难或者太长，告诉它"用更简单的话说，给小学生看的"
4. 把所有回答整理在一起，给每个部分起一个标题

**成功标准**
- [ ] 百科里有至少5个问题和对应的答案
- [ ] 至少有一个答案是你要求AI"用更简单的话再说一遍"后得到的
- [ ] 你能用自己的话，把其中一个问题的答案解释给同学或老师听
- [ ] 百科有标题（你的主题名字）

**快速完成者的挑战加餐**
加一页"有趣的冷知识"——让AI帮你找3个关于这个主题的、大多数人不知道的有趣事实，写进百科里。

#### 中学版（11-15岁）

**任务描述**
选一个你深入了解的话题，和AI一起创作一本"专题研究报告"，要求不只是堆砌事实，还要分析和比较。

步骤：
1. 确定主题，然后给这本报告设计一个目录结构（至少四个大章节，每章有1-2个子主题）
2. 用AI帮你填充每个章节的内容，但要求AI提供不同的观点或有争议的内容——不只是"XX是什么"，还要有"关于XX，有人认为……也有人认为……"
3. 每个章节完成后，用自己的知识检查一遍AI说的有没有明显错误（如果你不确定，让AI提供来源或者说明是基于什么信息）
4. 最后写一段"我的看法"——关于这个主题，你自己的观点是什么

**成功标准**
- [ ] 报告有完整的目录结构，四个以上章节
- [ ] 至少有一个章节包含"不同观点"的比较，不只是陈述事实
- [ ] 有一段自己写的"我的看法"，不是让AI写的
- [ ] 你能用3分钟给别人介绍这本报告，说清楚"我研究了什么、最有意思的发现是什么"

**快速完成者的挑战加餐**
把你的报告变成一个"测验"——从报告内容中提取5道问题，答案在报告里都能找到。把问题给同学来挑战，看他们答对几道。

### 挑战03：班级诗集网页 ⭐

**类别**：创意类
**目标阶段**：阶段二
**难度**：⭐️⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：60-90 分钟

**挑战目标**
制作一个展示班级成员写的诗的网页，让任何人打开都能读到这些诗。

**所需材料**
- 电脑 + AI工具（Claude Artifacts 或 Cursor）
- 收集班级同学写的诗（至少3首，可以在这个挑战之前单独用课时收集）

#### 小学版（6-10岁）

**任务描述**
做一个网页，把班级里大家写的诗放在上面，让任何人打开就能看到。

步骤：
1. 先收集3-5首同学写的诗（可以请同学在纸上写好，你再输入电脑）
2. 告诉AI："帮我做一个展示诗歌的网页，背景要好看，每首诗放在一个单独的卡片里，卡片上要显示诗的标题和作者名字"
3. 让AI生成网页，看看效果
4. 把真实的诗的内容更新到网页上（告诉AI"把第一首诗的内容换成：……"）
5. 让同学打开看，问他们"你能找到自己的诗吗？好不好看？"

**成功标准**
- [ ] 网页可以在浏览器里打开，显示正常
- [ ] 至少3首诗显示在网页上，每首都有标题和作者名
- [ ] 同学看了网页后，能找到自己的诗
- [ ] 你至少修改过一次——比如改了颜色、字体或者布局

**快速完成者的挑战加餐**
让网页上的每张诗卡片可以点击展开（默认只显示前两行，点开才显示全诗）。告诉AI"帮我改成点击才展开全诗的效果"。

#### 中学版（11-15岁）

**任务描述**
做一个完整的班级诗集网站，不只是把诗放上去，还要有分类、搜索和作者简介功能，让访问者有完整的阅读体验。

步骤：
1. 先设计网站结构：需要哪些页面？（诗歌列表页、诗歌详情页、按主题或作者筛选……）把这些写在纸上
2. 收集至少5首诗，还要收集每位作者的一句话简介
3. 告诉AI整体的设计需求，要求做出有导航的网站，而不只是一个页面
4. 做完基本功能后，找2个同学来测试：他们能找到某位同学的所有诗吗？有没有地方让他们困惑？
5. 根据测试反馈修改至少一个地方

**成功标准**
- [ ] 网站有至少两个不同的视图（比如：所有诗的列表 + 按作者筛选）
- [ ] 至少5首诗上线，每首有标题、内容、作者名
- [ ] 两位同学测试后，你记录了他们遇到的困难并做了修改
- [ ] 网站在手机或平板上打开也显示正常（让AI加入响应式设计）

**快速完成者的挑战加餐**
加一个"随机一首"按钮，每次点击就随机展示一首不同的诗，让访问者有惊喜感。

### 挑战04：个人成长日志工具

**类别**：创意类
**目标阶段**：阶段三
**难度**：⭐️⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：90-120 分钟

**挑战目标**
构建一个帮助自己记录每日成长的小工具，可以添加日志、查看历史记录，让每天的小进步都有地方留下痕迹。

**所需材料**
- 电脑 + ClaudeCode 或 OpenCode
- 纸笔（先画出应用结构草图）

#### 小学版（6-10岁）

**任务描述**
做一个每天都可以用来记录自己一件好事的工具，能保存你写的东西，下次打开还能看到。

步骤：
1. 先画出结构草图：工具需要几个部分？（写日志的地方 + 历史日志的列表）
2. 告诉AI你的结构，让它帮你做出第一个部分（写日志的功能）
3. 做出来后测试一下：写一条日志，看看能不能保存，再刷新页面看能不能看到
4. 如果测试发现问题，用你学过的"描述问题+期望结果"的方式告诉AI修复

**成功标准**
- [ ] 可以输入文字并保存一条日志
- [ ] 刷新页面后，之前写的日志还在（数据没丢）
- [ ] 能看到所有历史日志的列表，每条显示写的内容和日期
- [ ] 你自己用这个工具写了至少3条真实的日志

**快速完成者的挑战加餐**
加一个"今天我最开心的一件事"的固定提示语，让每次打开工具都自动显示这句话来提醒你。

#### 中学版（11-15岁）

**任务描述**
做一个多维度的成长日志应用，不只是记录事情，还能看出自己的进步趋势。

步骤：
1. 画出完整的结构图，包含：写日志模块、历史查看模块、统计模块（比如本周写了几条、最常记录的类型）
2. 让每条日志可以打上标签（比如：学习、围棋、人际关系、个人感受）
3. 实现统计功能：显示过去一周每种标签出现了几次
4. 做完所有模块后，自己测一遍完整流程，找出至少2个可以改进的地方并修复
5. 让一个同学试用，记录他们的反馈

**成功标准**
- [ ] 三个模块都能正常运行（写日志、历史查看、统计）
- [ ] 日志有标签功能，统计页面能按标签显示数量
- [ ] 完成过至少一次Debug：找到一个问题，用截图+描述告诉AI，修好后验证
- [ ] 同学试用后你记录了反馈，并说明哪条你选择改、哪条你选择不改（及理由）

**快速完成者的挑战加餐**
加一个"回顾"功能：每周日自动弹出"本周你写了X条日志，最多的标签是X，给自己一句话评价这周"，可以手动输入评价并保存。

### 挑战05：我的故事世界 ⭐

**类别**：创意类
**目标阶段**：阶段四
**难度**：⭐️⭐️⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：120-180 分钟

**挑战目标**
设计并构建一个展示虚构世界观的互动应用，包含世界地图、人物档案和故事片段，让任何人打开都能沉浸进这个世界。

**所需材料**
- 电脑 + ClaudeCode 或 OpenCode
- 纸笔（先画出世界地图草图和人物关系图）

#### 小学版（6-10岁）

**任务描述**
先用AI帮你创造一个虚构世界（有地方名、有居民、有规则），然后把这个世界做成一个网页，让别人可以点进去探索。

步骤：
1. 和AI对话，一起设计这个世界：它叫什么名字？有几个不同的地方？住着什么人或生物？有什么特别的规则？让AI帮你补充细节
2. 设计至少三个角色，每个角色有名字、家在哪、特别的能力或特点
3. 让AI帮你把这些内容做成一个网页，可以点击不同的地方来了解那里的信息
4. 在班上展示，看看同学有没有什么问题——"这个地方是做什么的？""这两个角色认识吗？"——把答案加进去

**成功标准**
- [ ] 虚构世界有名字，有至少3个不同的地方（有名字和描述）
- [ ] 有至少3个角色，每个都有名字和简介
- [ ] 网页可以打开，点击不同地方能看到对应的信息
- [ ] 至少一个同学看了以后问了一个你没有提前想到的问题，你把答案加进了网页

**快速完成者的挑战加餐**
给世界加一个短篇故事，让AI帮你写一个发生在这个世界里的小故事（500字左右），放进网页作为"世界传说"章节。

#### 中学版（11-15岁）

**任务描述**
构建一个可互动的世界观网站，不只是展示，还要有"探索"的感觉——用户可以沿着时间线查看历史、可以查看两个角色之间的关系。

步骤：
1. 做完整的世界设定文档（纸笔）：世界历史、地理、社会结构、主要角色及关系
2. 设计网站的信息架构：哪些内容放在哪里？用户怎么从一个地方导航到另一个地方？
3. 实现核心功能：地图展示、角色档案、至少两个角色之间的关系说明、时间线或历史事件列表
4. 在班级做一次展示，让两个同学来"探索"，观察他们点什么、在哪里停下来
5. 根据观察结果，改进最让人困惑的一处导航或内容

**成功标准**
- [ ] 网站有地图（可以是简单的示意图）、角色档案、时间线三个主要区域
- [ ] 角色之间有关系说明（比如"XX和XX是宿敌"）
- [ ] 两个同学的探索测试有记录（他们点了什么、在哪里停下来），你根据记录做了至少一处修改
- [ ] 能在3分钟内介绍你的世界，说清楚"最让你自豪的设计决定是什么，为什么"

**快速完成者的挑战加餐**
加一个"随机事件生成器"：点击按钮，AI随机生成一个发生在这个世界里的短故事，要符合世界的规则和已有角色的性格。

### 挑战06：AI学习助手

**类别**：实用类
**目标阶段**：阶段一
**难度**：⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：40 分钟

**挑战目标**
学会用AI作为学习工具——不是让AI帮你做作业，而是让AI帮你理解不懂的概念，练习"问清楚才能得到有用的回答"。

**所需材料**
- 电脑 + AI工具（Claude 或 ChatGPT 网页版）
- 最近学过但还不太懂的一个知识点（来自任何科目）

#### 小学版（6-10岁）

**任务描述**
找一个你在学校学过但还不太明白的知识点，和AI一起把它弄清楚。

步骤：
1. 先把你不懂的地方用自己的话说出来，发给AI。不用担心说得对不对，说清楚"我哪里不懂"就行
2. 看AI的回答。如果还是不明白，告诉它"用更简单的方式说，像在给小学生解释"
3. 让AI给你出2道关于这个知识点的题，你来回答，然后AI帮你看对不对
4. 如果做错了，让AI解释哪里做错了，再做一次类似的题

**成功标准**
- [ ] 你选了一个真实不懂的知识点（不是随便找一个）
- [ ] 你让AI至少解释了两遍，第二次要求了更简单的解释
- [ ] 你做了AI出的题，并且看了AI对错误的解释
- [ ] 你能用自己的话把这个知识点解释给同学听（不看AI的回答）

**快速完成者的挑战加餐**
让AI出一道"生活中的应用题"——把这个知识点和真实生活联系起来的题。比如学了加减法，就让AI出一道"你有多少零花钱，买了什么，还剩多少"的题。

#### 中学版（11-15岁）

**任务描述**
选一个比较复杂的概念（比如辩论里的逻辑谬误、数学里的概念、历史上的某个事件），用AI帮你深入理解并能向别人解释。

步骤：
1. 先自己写下"我对这个概念现在的理解"（哪怕只有一两句话，哪怕是错的）
2. 发给AI，让它评价你的理解哪里对、哪里不准确
3. 和AI进行至少三轮对话，每轮提一个新的问题——比如"那如果是这种情况呢？""它和另一个概念有什么不一样？"
4. 最后，在不看AI的情况下，用自己的话写一段这个概念的解释，写完后发给AI让它评分并指出还有哪些不足

**成功标准**
- [ ] 有"自己最初的理解"的记录
- [ ] 至少三轮追问，每轮问了不同角度的问题
- [ ] 最终用自己写的解释发给AI，AI确认理解基本正确
- [ ] 你能向一个同学解释这个概念，同学表示听懂了

**快速完成者的挑战加餐**
把你学到的知识做成一张"知识卡"——一张A4纸，用图示+关键词+一个例子的方式表达这个概念，贴在教室或者留在笔记本里。

### 挑战07：个人主页

**类别**：实用类
**目标阶段**：阶段二
**难度**：⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：45-60 分钟

**挑战目标**
制作一个展示自己爱好和作品的个人主页网页，完成后可以分享给同学或家人看。

**所需材料**
- 电脑 + AI工具（Claude Artifacts 或 Cursor）
- 你想放在主页上的内容（爱好、作品、一段自我介绍）

#### 小学版（6-10岁）

**任务描述**
做一个关于你自己的网页，让同学打开就能了解你是谁、你喜欢什么。

步骤：
1. 先写下要在主页上放什么：你的名字、你最喜欢的3件事、一句你想说的话、你做过的一件事
2. 告诉AI"帮我做一个个人主页，包含：名字显示在最上面，然后是我喜欢的3件事，然后是一张自我介绍，最后是我做过的一件事"
3. 把你真实的信息填进去（告诉AI"把名字改成我的名字，把爱好换成：……"）
4. 调整你觉得不好看的地方，比如颜色或者字体大小

**成功标准**
- [ ] 主页上有你的名字
- [ ] 有至少3项你的真实信息（爱好、介绍、作品等）
- [ ] 你至少改过一次——改了颜色、内容或者布局
- [ ] 你把主页展示给至少一个同学，他能说出"你最喜欢什么"

**快速完成者的挑战加餐**
加一个"给我发消息"的按钮，点击后弹出你的联系方式或者一段欢迎语。

#### 中学版（11-15岁）

**任务描述**
做一个真正有个人风格的主页，不只是放信息，还要有你做过的作品展示区域，设计要有自己的审美。

步骤：
1. 先想好你的个人主页"品牌"：你想给访客什么样的第一印象？简洁、活泼、正式还是有艺术感？
2. 做一个包含：个人简介、技能或爱好标签、作品集（至少两件你做过的东西）、联系或互动区域
3. 找一个同学，让他们不看你解释，自己浏览一遍，然后回答："你觉得这个人主要擅长什么？"——如果答案和你想展示的不一样，修改内容或布局
4. 调整直到同学的感知符合你的期望

**成功标准**
- [ ] 主页有至少四个区域（简介、爱好/技能、作品集、联系区域）
- [ ] 作品集里有至少两件真实的作品（可以是AI课上做的，也可以是其他科目的成果）
- [ ] 同学看完后对你的印象和你期望的基本一致（你记录了他们的反馈）
- [ ] 主页的设计风格是一致的，不是每个部分颜色和风格都乱

**快速完成者的挑战加餐**
加一个"暗黑模式"切换按钮，让访客可以在亮色和暗色主题之间切换。

### 挑战08：班级食谱大全

**类别**：实用类
**目标阶段**：阶段二
**难度**：⭐️⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：60-90 分钟

**挑战目标**
收录班级成员喜欢的食谱，做成一个可以搜索和浏览的网页食谱集，让任何人打开都能找到自己感兴趣的菜。

**所需材料**
- 电脑 + AI工具（Claude Artifacts 或 Cursor）
- 提前收集班级同学各自喜欢的食谱（可以是家常菜、零食做法等）

#### 小学版（6-10岁）

**任务描述**
把班级里大家最喜欢的3-5道菜做成一个网页食谱集。

步骤：
1. 提前从同学那里收集3-5道菜的名字和做法（可以让AI帮你整理步骤）
2. 告诉AI"帮我做一个食谱网页，每道菜有名字、谁提供的、需要什么食材、怎么做"
3. 把真实的食谱内容填进去
4. 让一个同学打开网页，问他"你能找到你最喜欢的菜吗？步骤清楚吗？"

- [ ] 网页上有至少3道不同的食谱
- [ ] 每道食谱有名字、提供者、食材和步骤

- [ ] 你改过至少一处——比如修改了格式或者补充了步骤

**快速完成者的挑战加餐**
加一个"今天吃什么"按钮，点击后随机展示食谱里的一道菜，给"选择困难症"同学一个随机推荐。

#### 中学版（11-15岁）

**任务描述**
做一个真正可用的班级食谱应用，有分类、搜索，还可以添加新食谱，不需要每次都找人修改代码。

步骤：
1. 设计功能列表：需要"按类型筛选"（比如：早餐、家常菜、零食）、"搜索食谱名字"、"添加新食谱"这三个核心功能
2. 画出结构草图，然后告诉AI整体需求，分模块实现
3. 让"添加食谱"功能真正可以工作——任何人输入食谱名字、食材、步骤，点击保存，就出现在列表里
4. 找两个同学测试：他们能用搜索找到某道菜吗？他们能添加一道新食谱吗？记录遇到的问题并修复

**成功标准**
- [ ] 有搜索功能，能按名字找到食谱
- [ ] 有按类型筛选（至少三种类型）
- [ ] 添加食谱功能可用——添加后立刻出现在列表里
- [ ] 两个同学测试后，你修复了至少一个他们遇到的问题

**快速完成者的挑战加餐**
加一个"评分"功能：每道食谱下面有1-5星的评分，点击打分后保存，食谱列表可以按评分排序。

3. 让计划按天显示——今天是几号，就显示今天该完成的任务

5. 用这个工具真实管理自己一周的学习，写下使用后的感受

**成功标准**
- [ ] 三个模块都能正常工作（输入、每日视图、统计）
- [ ] 任务有科目标签和预计时间
- [ ] 统计页面能正确计算完成率
- [ ] 你自己用了一周，并写下"这个工具帮了我什么、哪里用起来还不方便"

**快速完成者的挑战加餐**
加一个"建议"功能：当你把这周所有任务都输入后，工具自动分析哪天任务最多，给出一句话建议（比如"你周三计划了3小时，可能有点多，建议把XX移到周四"）。

### 挑战10：中心活动日历 ⭐

**挑战目标**

完整走一遍产品设计流程：从调研用户需求，到设计方案，到构建，到测试，最终交付一个真正被用户使用的活动日历。

步骤：
1. 访谈至少3个不同的用户（同学、老师、管理人员），问他们最需要什么功能，现在怎么知道活动信息，最不方便的是什么

**成功标准**
- [ ] 有完整的调研记录（访谈了谁、说了什么、你决定做什么）
- [ ] 需求文档清楚说明了核心功能和放弃的功能及理由
- [ ] 活动管理功能可用（可以添加、修改活动）
- [ ] 两轮用户测试都有记录，每轮都有根据反馈的修改
- [ ] 至少一个用户表示"这个我以后会真的用"

**快速完成者的挑战加餐**
加一个"通知"功能：活动当天在首页显示一个提醒横幅"今天有活动：XX，地点：XX，时间：XX"，让访问者一眼就看到。

#### 小学版（6-10岁）

**任务描述**
用AI帮你写一篇关于本妙坊的介绍，让新来的同学读了就知道这里是什么、有什么课、为什么好玩。

步骤：
1. 先想好要介绍哪些方面：这里有什么课？有什么特别的地方？同学在这里最喜欢做什么？
2. 告诉AI"帮我写一段介绍本妙坊未来成长中心的文字，给刚来的同学看，要写得有意思，让他们觉得这里很好玩"
3. 读一读AI写的，把不准确的地方改掉（比如AI可能不知道具体的科目名字，告诉它正确信息）
4. 让你认识的一个新同学（或者假装是新同学的老同学）读一读，看他们有没有疑问

**成功标准**
- [ ] 介绍里有本妙坊的真实信息（不是AI乱编的）
- [ ] 你改过AI写的内容——纠正了至少一处错误或不准确的地方
- [ ] 另一个同学读了以后，能说出本妙坊有哪些课
- [ ] 介绍读起来是有趣的，不是干巴巴的清单

**快速完成者的挑战加餐**
**阶段二加餐**：把这段介绍做成一个网页，加上不同的颜色和排版，让新同学打开网页就能了解本妙坊。

#### 中学版（11-15岁）

**任务描述**
做一个完整的"新生指南"网页，不只是介绍，还要帮助新同学快速融入——知道去哪里问问题、有什么活动可以参与、同学之间怎么互动。

步骤：
1. 先访谈1-2个已经在这里学了一段时间的同学："你刚来的时候，最希望提前知道什么？"
2. 根据访谈，确定新生指南需要包含哪些内容
3. 用AI帮你撰写和组织内容，做成有多个部分的网页（比如：学校介绍、科目说明、常见问题、联系方式）
4. 找一个真的是新同学或者模拟新同学的人试用，看他们能否找到需要的信息

**成功标准**
- [ ] 访谈了至少一个同学，记录了他们希望新生知道什么
- [ ] 网页有至少三个不同部分，内容都是真实的
- [ ] 有常见问题（FAQ）部分，至少包含三个真实的新生可能有的问题
- [ ] 一个"新同学"试用后，能找到他们想要的信息

**快速完成者的挑战加餐**
加一个互动问答：提供几个问题让新同学回答（比如"你喜欢什么科目？"），根据他们的答案，给出个性化的建议（比如"既然你喜欢XX，你可以先去了解这个活动"）。

### 挑战11：给家人的AI信

**类别**：社交类
**目标阶段**：阶段一
**难度**：⭐️（1-5级，⭐️最简单）
**预计时间**：30-40 分钟

**挑战目标**
用AI帮忙写一封有感情、有内容的信给家人，学会把"想说的话"转化为AI可以帮助润色的提示，最终产出一封真正可以发出去或打印出来的信。

**所需材料**
- 电脑 + AI工具（Claude 或 ChatGPT 网页版）
- 想清楚：这封信是给谁的？想说什么？

#### 小学版（6-10岁）

**任务描述**
给家里最常陪你的一个大人写一封信，告诉他们你最近在做什么、学到了什么好玩的事、或者有什么想感谢他们的话。

步骤：
1. 先用一句话说清楚：这封信是写给谁的，想说什么主要的话
2. 告诉AI："帮我写一封信，写给我的[关系，比如爸爸]，我想说[你想说的话]。字数不要太多，大概100字，要像是真正我说话的感觉，不要太正式"
3. 读一读AI写的，看有没有听起来不像是你说的话——如果有，告诉AI"这句话不像我会说的，改成更自然的方式"
4. 改到满意为止，然后把信抄写或打印出来

**成功标准**
- [ ] 信里有你真实想说的内容（不是随便让AI编的）
- [ ] 你改过AI写的内容，说出了"这里不对，改成……"
- [ ] 最终的信你愿意真的给那个家人看
- [ ] 你能说出这封信里你最喜欢的一句话是什么

**快速完成者的挑战加餐**
写第二封信：给另一个你想感谢或想关心的人，这次先自己写一段草稿，再让AI帮你润色，比较AI润色前后有什么不同。

#### 中学版（11-15岁）

**任务描述**
写一封更有深度的信，不只是日常分享，而是认真说一件想说很久但没有机会开口的事——比如一次让你成长的经历、一件想感谢但从没说出来的事，或者对未来的一个想法。

步骤：
1. 确定主题：这封信的核心是什么？把你真正想表达的一段话先写在纸上（不用AI，先写自己的版本）
2. 再把你的草稿发给AI，让它帮你"润色而不改变你的意思"——要求保持你的声音，只改表达方式
3. 对比AI的版本和你的版本，选出每段里你更喜欢的那个，然后组合成最终版
4. 大声朗读最终版，感受一下每句话读出来是否自然，不自然的地方继续调整

**成功标准**
- [ ] 有你自己写的草稿，不是直接让AI写的
- [ ] AI版本和你的版本有对比，你做出了取舍
- [ ] 信里有一段完全是你自己写的，没有被AI改动
- [ ] 你可以大声朗读这封信，每句话读出来都是你想说的感觉

**快速完成者的挑战加餐**
把这封信的核心想法变成一首短诗（8行以内），让AI根据信的内容帮你写，然后你修改，看诗歌的表达和散文有什么不同。

### 挑战12：班级公告板

**类别**：社交类
**目标阶段**：阶段二
**难度**：⭐️⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：60-90 分钟

**挑战目标**
做一个班级可以一起看的公告网页，让同学打开就能知道班里最新的通知和活动。

**所需材料**
- 电脑 + AI工具（Claude Artifacts 或 Cursor）
- 几条真实的班级通知（可以是假设的）

#### 小学版（6-10岁）

**任务描述**
做一个班级公告板网页，让老师或班委可以更新，同学打开就能看到最新通知。

步骤：
1. 想清楚公告板需要什么：通知的标题、内容、日期，有没有"重要"标记
2. 告诉AI"帮我做一个班级公告板网页，每条公告显示标题、日期和内容，最新的显示在最上面"
3. 把3条真实（或模拟）的班级通知填进去
4. 让同学打开看，问他们"你看到了哪条最近的通知？"

**成功标准**
- [ ] 网页显示至少3条通知，每条有标题、日期和内容
- [ ] 通知按日期排列，最新的在上面
- [ ] 同学打开后能立刻找到最新的通知
- [ ] 你改过至少一次设计或内容

**快速完成者的挑战加餐**
加一个"重要"标记功能：某些通知可以被标为"重要"，重要通知在页面顶部用不同颜色突出显示。

#### 中学版（11-15岁）

**任务描述**
做一个真正可用的班级公告管理系统，不只是展示，还可以添加、编辑通知，并且有简单的分类功能。

步骤：
1. 设计功能：展示公告列表、添加新公告、按类别筛选（比如：活动、通知、作业提醒）
2. 分模块构建，先做展示，再做添加功能
3. 添加功能要真正可用：填写标题、内容、类别，点提交后出现在列表里
4. 找两个同学测试：他们能找到某类通知吗？能添加一条新通知吗？记录问题并修复

**成功标准**
- [ ] 展示、添加、分类筛选三个功能都可用
- [ ] 添加通知后立刻显示在列表里，不需要刷新
- [ ] 两个同学测试后，修复了至少一个问题
- [ ] 如果不小心提交了空表单，有提示告诉用户"请填写标题"

**快速完成者的挑战加餐**
加一个"置顶"功能：管理员可以把某条通知置顶，置顶的通知始终显示在所有通知最上面，有特殊样式。

### 挑战13：同学推荐系统

**类别**：社交类
**目标阶段**：阶段三
**难度**：⭐️⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：90-120 分钟

**挑战目标**
构建一个同学互相推荐书、电影、游戏的小工具，任何人可以添加推荐，任何人可以浏览和搜索。

**所需材料**

#### 小学版（6-10岁）

**任务描述**
做一个"好东西推荐"工具，同学可以在上面分享自己觉得好看/好玩的书或电影，其他人可以浏览。

步骤：
1. 画出结构：需要哪些部分？（添加推荐的表单 + 推荐列表）

3. 再做"添加推荐"功能，让任何人可以输入名字、类型（书/电影/游戏）、推荐理由

加一个"我也想看"按钮，点击后这条推荐下面的计数加一，显示"X个人感兴趣"。

做一个功能完整的推荐系统，支持搜索、按类型筛选、评分，还有"推荐人"功能——知道是谁推荐的。

1. 设计完整结构：推荐列表模块（含搜索和筛选）、添加推荐模块（含推荐人信息）、评分模块
2. 分模块构建，每个模块做完后测试
3. 搜索功能要能按关键词搜索（在名字或推荐理由里搜索）
4. 做完后进行系统测试：列出至少5个可能出问题的操作，逐一测试，记录发现的问题并修复

**成功标准**
- [ ] 搜索、筛选、添加、评分四个功能都可用
- [ ] 系统测试记录有至少5个测试场景，发现的问题都已修复
- [ ] 推荐人字段有效——列表里能看到每条推荐是谁推荐的
- [ ] 完成Debug记录（找到并修复过至少一个问题，有截图或描述记录）

**快速完成者的挑战加餐**
加一个"随机推荐"功能：点击按钮，随机展示一条推荐，并显示"如果你喜欢，点赞；如果不感兴趣，继续随机"，让用户可以一直随机直到找到感兴趣的。

### 挑战15：AI出题游戏

**类别**：游戏类
**目标阶段**：阶段一
**难度**：⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：40-50 分钟

**挑战目标**
让AI帮你出一套关于某个主题的问答题，然后在班上举办一场小测验，成为出题者，而不是只会做题的人。

**所需材料**
- 电脑 + AI工具（Claude 或 ChatGPT 网页版）
- 一张纸记录AI出的题目（用于班上展示）

#### 小学版（6-10岁）

**任务描述**
选一个你很感兴趣的主题，让AI出5道问答题，难度适合你的同班同学，然后在班上当一次"主持人"出题考大家。

步骤：
1. 选主题（比如：动物、体育、食物、历史故事）
2. 告诉AI"帮我出5道关于[主题]的问答题，难度适合10岁的孩子，每道题有一个正确答案，答案写在题目下面"
3. 检查这些题：每道题你自己先回答一遍，看答案对不对，如果有错的告诉AI"第X题答案不对，正确答案是……"
4. 在班上举办问答，你来出题，让同学举手回答，记录谁回答对了几道

**成功标准**
- [ ] AI出了5道题，你每道都检查过答案
- [ ] 你纠正了AI至少一处错误（或者问了AI"你确定这个答案对吗？"）
- [ ] 在班上出了题，至少3个同学参与回答
- [ ] 你能说出哪道题最难、哪道最容易，和你预期的一不一样

**快速完成者的挑战加餐**
让AI出一道"连环题"——第一道题的答案是第二道题的线索，一共三道连环，最后一道揭开一个有趣的秘密。

#### 中学版（11-15岁）

**任务描述**
做一套真正有深度的问答题，要有不同难度、不同类型（选择题、判断题、简答题），并且每道题都附上解析，让回答错了的人也能学到东西。

步骤：
1. 选一个你真正了解的主题（比如围棋规则、某本书、某段历史）
2. 让AI出10道题，要求：3道简单（单一答案）、4道中等（需要理解）、3道难（需要推理）
3. 每道题你自己先做一遍，然后对照AI的答案。对AI写的解析提出质疑："这个解释够清楚吗？能举个例子吗？"
4. 在班上组织一场正式的问答赛：有分数、有计时、有最终排名
5. 赛后，把你认为出得最好的3道题分享给同学，说明"为什么这道题设计得好"

**成功标准**
- [ ] 10道题有三个难度层次，类型不全是选择题
- [ ] 每道题都有你写的或改过的解析
- [ ] 你改过AI出的题或答案至少3次
- [ ] 在班上成功举办了问答赛，有完整的计分
- [ ] 能说清楚"一道好的问答题有什么特点"

**快速完成者的挑战加餐**
用AI帮你写一道"陷阱题"——看起来答案是A，但仔细想是B，并且写一段解释"为什么容易答错、正确答案背后的逻辑是什么"。

4. 测试排班是否公平：统计每个人值日的次数，看是否均衡

**成功标准**
- [ ] 排班逻辑基于真实讨论的公平规则
- [ ] 可以自动生成一个月的排班
- [ ] 统计显示每人值日次数基本均衡（误差不超过1次）
- [ ] 支持手动修改某天的安排
- [ ] 班主任或班委试用后，反馈"这比手动排班省事"

**快速完成者的挑战加餐**
加"请假处理"：某同学请假当天，工具自动标记并提示需要人顶替，管理员可以指定顶替人并更新安排。

### 挑战23：迷你图书馆系统

**类别**：真实问题类
**目标阶段**：阶段四
**难度**：⭐️⭐️⭐️⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：180-240 分钟

**挑战目标**
为本妙坊打造一个完整的图书管理应用，覆盖书目管理、借还记录、搜索推荐、统计报告，真正解决图书角管理混乱的问题。

**所需材料**
- 电脑 + ClaudeCode 或 OpenCode
- 与图书角管理员（老师或学生）深度访谈

#### 小学版（6-10岁）

**任务描述**
做一个比挑战20更完整的图书管理工具，加入搜索功能和简单的图书推荐。

步骤：
1. 先用挑战20的借还记录作为起点（或重新开始，用更好的结构）
2. 加入图书搜索：可以按书名或分类查找书
3. 加入图书推荐：在书的详情页，显示"其他同学借了这本书的人也借了：XX"
4. 让3个同学来用，记录他们的反馈并修改

**成功标准**
- [ ] 借还功能正常工作
- [ ] 搜索可以找到书
- [ ] 每本书有分类，可以按分类筛选
- [ ] 3个同学使用后，你根据反馈做了至少2处改进

**快速完成者的挑战加餐**
加一个"我的书单"功能：同学可以把感兴趣的书加入自己的"想读清单"，等书被归还后可以优先借阅。

#### 中学版（11-15岁）

**任务描述**
完整走阶段四的产品设计流程，从深度用户调研，到功能设计，到多模块构建，到两轮真实用户测试，最终交付一个真正可以被本妙坊使用的图书管理系统。

步骤：
1. 深度访谈：图书角管理员（老师或学生）、图书角的常客同学、偶尔来的同学——每类用户的需求不同
2. 写完整的需求文档，包含核心功能、次要功能（有时间再做）、明确放弃的功能
3. 分阶段构建：先完成核心功能（书目+借还），再加附加功能（搜索+统计）
4. 第一轮用户测试：分别让管理员和普通同学测试，关注他们各自的使用场景
5. 修改后第二轮测试，确认改进有效
6. 写一份"交接文档"：说明这个系统有哪些功能、怎么用、如果遇到问题怎么解决

**成功标准**
- [ ] 有三类用户的访谈记录，需求文档清晰
- [ ] 核心功能（书目管理、借还、搜索）全部正常工作
- [ ] 有统计模块，显示图书角的使用情况
- [ ] 两轮用户测试都有记录，每轮都有基于反馈的修改
- [ ] 有交接文档，另一个人读了文档后能使用这个系统
- [ ] 图书角管理员表示"这个我会真的用"

**快速完成者的挑战加餐**
加一个"阅读活动"功能：设置一个班级阅读目标（比如本月读30本书），追踪班级整体进度，当达成目标时有庆祝提示。

### 挑战24：成长记录本 ⭐

**类别**：真实问题类
**目标阶段**：阶段四
**难度**：⭐️⭐️⭐️⭐️（1-5级，⭐️最简单）
**预计时间**：120-180 分钟

**挑战目标**
制作一个记录每位学生成长里程碑的工具，适合教师或助教使用，让每个孩子的重要时刻都有地方被记录和分享。

**所需材料**
- 电脑 + ClaudeCode 或 OpenCode
- 先访谈一位老师，了解他们希望记录什么、在什么情况下使用

#### 小学版（6-10岁）

**任务描述**
做一个"班级成长相册"工具，老师或助教可以给每个同学记录一个值得纪念的时刻（比如第一次独立完成作品、在比赛中表现很好、帮助了同学），同学和家长可以查看。

步骤：
1. 先问一位老师："你平时会帮同学记录哪些成长的时刻？你更愿意怎么记录？"
2. 根据回答设计工具：学生列表 + 给每个学生添加成长记录的功能 + 按学生查看记录的功能
3. 分模块构建，先做学生列表，再做添加记录功能
4. 让老师来试用，看他们能顺利添加一条记录

**成功标准**
- [ ] 有学生列表，点进去可以看到该学生的所有记录
- [ ] 可以给学生添加成长记录（有日期、描述、可选的类型标签）
- [ ] 老师试用后能顺利添加记录，不需要你帮忙操作
- [ ] 界面对老师友好，不需要任何技术知识就能用

**快速完成者的挑战加餐**
加"家长视图"：给每个学生生成一个简单的"分享链接"，家长通过这个链接可以看到孩子的成长记录，但不能修改。

#### 中学版（11-15岁）

**任务描述**
完整走产品设计流程，做一个真正让老师和家长都满意的学生成长记录系统，在 Demo Day 上展示给家长看。

步骤：
1. 访谈至少2位老师（或助教）和2位家长，了解各自的需求——老师关注什么？家长想看到什么？两者有没有不同？
2. 根据调研写需求文档，特别注意"老师视角"和"家长视角"的区别
3. 设计两种不同的视图：教师管理视图（添加记录、管理学生）、家长/只读视图（按时间线查看孩子的记录）
4. 分模块构建，每完成一个模块测试
5. 分别让一位老师和一位家长测试，各自的使用场景
6. 在 Demo Day 上展示——让来的家长现场查看（假设的）自己孩子的成长记录

**成功标准**
- [ ] 有老师和家长的访谈记录，两者需求的异同被分析过
- [ ] 两种视图都可以正常工作，功能不一样
- [ ] 老师和家长各自测试后，都反馈"这个设计考虑到了我的需求"
- [ ] Demo Day 展示时，家长现场看到记录后有正面反馈
- [ ] 你能解释"为什么老师视图和家长视图设计不同"的理由

**快速完成者的挑战加餐**
加"学期总结生成"功能：对每个学生，点击一个按钮，系统自动把这学期的所有成长记录整理成一段话，作为学期总结的草稿，老师可以进一步修改。
