# Final Project: 从零构建你的第一个真实产品
# Final Project: Build Your First Real Product from Scratch

> **这是什么 / What This Is**
>
> 这是整个课程的综合实战练习。你将把 4 讲里学到的所有内容整合起来，用完整的专业工作流，从一个想法出发，独立完成一个真实的 Web 应用。
>
> This is the capstone of the entire course. You will integrate everything from all 4 lectures and build a complete real web app from idea to deployment using professional workflow.

---

## 项目要求 / Project Requirements

### 你要做什么

选择以下**任意一个项目**（或自己提出一个同等复杂度的想法）：

| 选项 | 项目名 | 功能描述 |
|------|-------|---------|
| A | **班级公告板** | 老师可以发布公告，学生可以查看；公告支持标题+正文+重要程度标记；可以按日期排序 |
| B | **投票小工具** | 创建一个投票问题，设置多个选项；用户可以投票（每人一次）；实时显示投票结果图表 |
| C | **单词记忆卡** | 添加单词和释义，形成卡片；支持翻转卡片查看答案；记录每张卡片的"记住了/没记住"状态 |
| D | **自选项目** | 你自己提出一个想法，需要老师/同学确认范围合理后开始 |

### 完成标准

你的项目必须满足：
- [ ] 基本功能完整（增删查）
- [ ] 界面美观，在手机和电脑上都能正常显示
- [ ] 有一个 AGENTS.md 文件
- [ ] Git 有至少 5 次有意义的提交记录
- [ ] 完成了安全审查步骤（见下方）
- [ ] 能在浏览器里运行并展示

## 完整工作流 / Full Workflow

按照这 6 个阶段进行，**每个阶段都有要提交的成果**。

---

### 阶段 0：立项规划（约 30 分钟）

**你的任务：**

1. 选择你的项目
2. 把项目拆成功能模块，填写下表：

```
项目名称：_______________
选用工具：_______________（建议：Bolt.new / Lovable / Cursor）

功能模块拆解：
┌─────────────────────────────────┐
│ 模块名    │ 具体功能              │ 优先级   │
│──────────│────────────────────-│─────────│
│ 核心功能 │                      │ 必须有   │
│ 界面展示 │                      │ 必须有   │
│ 数据存储 │                      │ 必须有   │
│ 增强功能 │                      │ 有时间做 │
└─────────────────────────────────┘

技术栈（让 AI 帮你决定也可以）：_______________
```

3. 把功能模块进一步拆成**原子任务**（每个任务 AI 一次对话能完成的大小）：

```
示例（投票小工具）：
任务 1: 搭基础架构，React + Tailwind，建好文件结构
任务 2: 做"创建投票"表单（标题 + 最多4个选项）
任务 3: 做"投票列表"页面，显示所有投票
任务 4: 做"投票详情"页，展示各选项和投票按钮
任务 5: 实现投票逻辑（每用户只能投一次，用 localStorage 记录）
任务 6: 添加投票结果图表（用进度条显示百分比）
任务 7: 响应式布局，确保手机端正常
任务 8: 界面美化和动画效果
```

**阶段 0 交付物：** 填完的项目规划表 + 原子任务清单

---

### 阶段 1：写 AGENTS.md（约 15 分钟）

在你的项目里创建 `AGENTS.md`，模板如下（根据你的项目修改）：

```markdown
# 项目名称

## 项目简介
[一句话描述：这个项目是什么、给谁用的]

## 技术栈
- 前端：[React/Vue/Vanilla JS] + [Tailwind/Bootstrap]
- 数据存储：localStorage（纯前端）/ [其他]
- 部署：[Vercel/Netlify/直接在浏览器里跑]

## 设计风格
- 主色调：[你的颜色，如 #3B82F6 蓝色]
- 风格：[现代简洁 / 扁平化 / 其他]
- 目标用户：[中学生 / 老师 / 其他]

## 编码规范
- 组件化开发，每个功能一个组件文件
- 中文注释（因为这是教学项目）
- 不使用外部数据库，数据存在 localStorage

## 禁止事项
- 不要安装未经确认的新依赖
- 不要删除已有文件，只修改或新增
- 不要使用 CSS 内联样式，只用 Tailwind 类名

## 当前进展
[每个阶段开始时更新这里]
```

**阶段 1 交付物：** 完整的 AGENTS.md 文件

---

### 阶段 2：搭基础架构（约 30 分钟）

**你的提示词模板（第一次对话）：**

```
"我在做一个[项目名]。请帮我搭建基础架构：
技术栈：[你的技术栈]
项目结构要求：
  - 清晰的文件夹结构
  - 主页面框架（顶部导航 + 主内容区 + 底部）
  - 设计风格：[你的风格描述]

不需要实现任何具体功能，只需要：
1. 基础 HTML/CSS/JS 框架（或 React 基础结构）
2. 空白的主页面，有正确的布局
3. 颜色和字体设置好

请先给我展示文件结构，再生成代码。"
```

**完成后：**
```bash
git init
git add .
git commit -m "阶段2: 搭建基础架构"
```

**阶段 2 交付物：** 能在浏览器打开的空框架 + 第一次 git commit

---

### 阶段 3：逐功能开发（每个功能 30-45 分钟）

按照阶段 0 的任务清单，**一次做一个任务**。

**每个功能的标准流程：**

```
1. 写提示词（参考 Lecture 3 的模板）
   - 包含：背景 + 具体要求 + 已有的结构信息

2. AI 生成代码

3. 在浏览器里测试
   - 正常情况能不能跑通？
   - 边界情况（空表单、超长文字、快速点击）怎样？

4. 迭代（如果不对）
   - 描述哪里不对
   - 让 AI 修改

5. 确认 OK 后，git commit
   git commit -m "完成任务X: [功能名称]"
```

**每个功能完成后，填写记录表：**

```
功能名称：_______________
使用的提示词（关键部分）：
_______________

遇到的问题：_______________
解决方式：_______________
迭代了几次才满意：_______________
```

**阶段 3 交付物：** 所有核心功能完成 + 每个功能有独立的 git commit + 功能记录表

---

### 阶段 4：界面美化与收尾（约 30 分钟）

这个阶段专注"让它好看、好用"：

**检查清单：**
- [ ] 手机端（小屏幕）显示正常
- [ ] 所有按钮有 hover 效果
- [ ] 加载/操作时有反馈（按钮变灰、显示"保存中..."）
- [ ] 空状态有友好提示（"还没有数据，点击添加第一条"）
- [ ] 错误情况有提示（表单验证、输入为空等）

**提示词示例：**
```
"请检查整体界面，改进以下方面：
1. 确保在 375px 宽度（iPhone）下布局正常
2. 所有交互元素加 hover/active 效果
3. 添加适当的过渡动画（0.2s 即可）
4. 空状态页面加友好的占位文字和图标"
```

```bash
git commit -m "阶段4: 界面美化与收尾"
```

**阶段 4 交付物：** 美化完成的完整项目 + git commit

---

### 阶段 5：安全审查（约 20 分钟）

这是 Lecture 4 的实践。即使是个人小项目，也要养成审查习惯。

**让 AI 做安全审查：**

```
"请对这个项目做一次安全审查，检查以下方面：
1. 用户输入的数据有没有经过验证和清理（防止 XSS 攻击）？
2. localStorage 里存储的数据有没有敏感信息？
3. 有没有可能被恶意用户利用的接口或功能？
4. 代码里有没有写死的密码、密钥或敏感信息？
请给出发现的问题和修复建议。"
```

**记录审查结果：**

```
发现的安全问题：
1. _______________（是否修复：是/否）
2. _______________（是否修复：是/否）

AI 没发现但我注意到的问题：_______________

这个项目如果有真实用户，我最担心的安全风险是：_______________
```

```bash
git commit -m "阶段5: 安全审查与修复"
```

**阶段 5 交付物：** 安全审查记录 + 修复后的 git commit

---

### 阶段 6：展示与反思（约 20 分钟）

**最终演示（3 分钟）：**

准备一个简短的演示，包含：
1. 你的项目是什么，解决了什么问题（30秒）
2. 演示核心功能（90秒）
3. 分享一个让你印象最深的 Vibe Coding 时刻（好的或坏的）（60秒）

## 最终反思报告

项目完成后，写一份简短的反思（不超过 500 字），回答以下问题：

**关于 Vibe Coding 工作流：**
1. 规划阶段花的时间值得吗？为什么？
2. AGENTS.md 有没有帮助 AI 做出更好的选择？
3. 频繁 git commit 有没有帮到你？有没有回滚过？

**关于提示词：**
4. 哪一次提示词效果最好？它有什么特点？
5. 哪一次 AI 完全没理解你的意思？你是怎么解决的？

**关于风险：**
6. 安全审查找到了什么问题？你觉得严重吗？
7. 你现在觉得 Vibe Coding 适合做哪类项目？不适合做哪类？

**关于自己：**
8. 和课程开始之前相比，你对 Vibe Coding 的看法有什么变化？
9. 你认为自己接下来最需要继续学习什么？

## 评分标准（供教师参考）

| 维度 | 满分 | 说明 |
|------|------|------|
| 功能完整性 | 30 | 核心功能是否都实现了 |
| 工作流执行 | 20 | 有没有按 5 阶段走，git commit 是否规范 |
| AGENTS.md 质量 | 10 | 是否清晰、有实际指导价值 |
| 提示词记录 | 15 | 记录是否完整，有没有反思 |
| 安全审查 | 15 | 是否认真做了，发现了什么 |
| 反思报告 | 10 | 思考深度，不是字数 |

**加分项：**
- 自选项目（有创意）：+5

- 帮助同学解决了问题：+3
